## Model / Domain classes (key ones used by the WPF UI)
- DockWorkspace — docking/layout workspace manager
- AssemblyTreeModel — model for the assembly tree pane
- LoadedAssembly — represents an opened assembly
- DecompilerSettings — decompiler configuration
- SettingsService / SettingsServiceBase — app settings persistence/service
- LanguageService — language selection/service used by the UI
- Tree node classes under TreeNodes/ (e.g., AssemblyTreeNode, ResourceTreeNode, various Metadata tree nodes)
- MessageBus — internal message/event bus used by the UI

## Notes
- The WPF app follows MVVM-like separation: Views under `ILSpy/Views` and `ILSpy/Controls` bind to view-models in `ICSharpCode.ILSpy.ViewModels`.
- Domain models and services are spread across `ICSharpCode.ILSpyX`, `ICSharpCode.Decompiler`, and `ILSpy` projects (e.g., `LoadedAssembly`, `DecompilerSettings`).
